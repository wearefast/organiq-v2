# Feature: Content

## Overview

Content pieces represent SEO content assets — either briefs (outlines for writers) or full articles. They are generated by workflow agents and stored per-project, linked to keywords and workflow runs.

## Key Files

| File | Purpose |
|------|---------|
| `server/src/features/content/content.controller.ts` | REST API under `projects/:projectId/content` |
| `server/src/features/content/content.service.ts` | CRUD + bulk create + stats |
| `server/src/features/content/content.module.ts` | NestJS module |
| `frontend/src/features/content/services/content.service.ts` | Frontend API service |
| `frontend/src/app/(dashboard)/workspaces/[wId]/projects/[pId]/content/page.tsx` | Content list/editor page |

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/projects/:projectId/content` | List all content pieces |
| `GET` | `/projects/:projectId/content/stats` | Aggregate stats by status/type |
| `GET` | `/projects/:projectId/content/:id` | Get single content piece |
| `POST` | `/projects/:projectId/content` | Create a content piece |
| `POST` | `/projects/:projectId/content/bulk` | Bulk create content pieces (used by agents) |
| `PATCH` | `/projects/:projectId/content/:id` | Update content piece |
| `PATCH` | `/projects/:projectId/content/:id/status` | Update content status |

## Data Model

### content_pieces table

| Column | Type | Description |
|--------|------|-------------|
| id | UUID | Primary key |
| projectId | UUID | FK → projects |
| keywordId | UUID | FK → keywords (nullable) |
| workflowRunId | UUID | FK → workflow_runs (nullable) |
| type | enum | `brief` or `article` |
| status | enum | draft, review, approved, published |
| title | text | Content title / headline |
| briefData | jsonb | Structured brief (outline, talking points, E-E-A-T signals, target keyword data) |
| articleData | jsonb | Full article content (markdown body, meta description, headings) |
| scores | jsonb | Quality scores (readability, SEO, citability, content length) |
| wordCount | integer | Total word count |

### Indexes
- `content_pieces_project_idx` — Fast project filtering
- `content_pieces_status_idx` — Filter by project + status

## Content Lifecycle

```
brief (draft) → brief (review) → brief (approved) → article (draft) → article (review) → article (approved) → published
```

### Brief Generation (content-brief agent)
- Input: Approved keyword + topical map context
- Output: `briefData` with outline, target word count, E-E-A-T signals, competing content analysis
- Enforced contract: `paaQuestions`, `internalLinks`, and `ctaRecommendations` are persisted as structured object arrays, not string lists; malformed Step 16 outputs now fail validation before artifact persistence

### Article Generation (content-article agent)
- Input: Approved brief
- Output: `articleData` with full markdown body, meta description, internal linking suggestions

## Scoring Framework

Content is scored across 4 dimensions via rubrics in `server/src/prompts/scoring/`:

| Dimension | Rubric File | Score Range |
|-----------|-------------|-------------|
| Readability | `readability.rubric.md` | 0-100 |
| SEO Quality | `seo-quality.rubric.md` | 0-100 |
| AI Citability | `citability.rubric.md` | 0-100 |
| Content Length | `content-length.rubric.md` | 0-100 |

Scores are stored in the `scores` JSONB column on the content piece.
